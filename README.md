ระบบรับเรื่องร้องเรียนและจัดการปัญหาของนักศึกษา (Complaint Management System) พัฒนาขึ้นเพื่อช่วยให้สถาบันการศึกษาสามารถรับเรื่อง แจ้งเตือน และจัดการปัญหาต่างๆ ของนักศึกษาได้อย่างเป็นระบบ รวดเร็ว และสามารถติดตามสถานะได้อย่างโปร่งใส

## 🚀 เทคโนโลยีที่ใช้ (Tech Stack)

### Frontend (หน้าบ้าน)
* **Next.js (App Router)**: Framework หลักสำหรับ React ที่รองรับ Server-Side Rendering (SSR)
* **React**: ไลบรารีสำหรับสร้าง User Interface
* **Tailwind CSS**: Utility-first CSS framework สำหรับออกแบบ UI ที่สวยงามและ Responsive
* **TypeScript**: เพื่อการเขียนโค้ดที่ปลอดภัยและลดข้อผิดพลาด (Type-safety)

### Backend (หลังบ้าน)
* **Next.js API Routes**: จัดการ API ภายในตัว Framework
* **Prisma (ORM)**: เครื่องมือจัดการฐานข้อมูลที่ช่วยให้เขียนโค้ดติดต่อฐานข้อมูลและจัดการ Schema ได้อย่างมีประสิทธิภาพ
* **NextAuth.js**: ระบบจัดการการยืนยันตัวตน (Authentication) และการจัดการสิทธิ์ (Role-Based Access Control) แบบปลอดภัย

### Database (ฐานข้อมูล)
* **MySQL**: ระบบจัดการฐานข้อมูลเชิงสัมพันธ์ (Relational Database) สำหรับเก็บข้อมูลผู้ใช้, เรื่องร้องเรียน, และคณะ/สาขา

## ✨ ฟีเจอร์หลักของระบบ (Key Features)

### 1. ระบบจัดการสิทธิ์ผู้ใช้งาน (Role-Based Access Control)
แบ่งผู้ใช้งานออกเป็น 4 ระดับ เพื่อความปลอดภัยและขอบเขตการเข้าถึงข้อมูล:
* **นักศึกษา (Student)**: ผู้แจ้งเรื่องร้องเรียน
* **อาจารย์ที่ปรึกษา (Advisor)**: ดูแลและติดตามเคสของนักศึกษาในที่ปรึกษา
* **ผู้ดำเนินการ (Operator)**: เจ้าหน้าที่ผู้รับผิดชอบรับเรื่องและแก้ไขปัญหาตามที่ได้รับมอบหมาย
* **ผู้ดูแลระบบ (Admin)**: จัดการภาพรวมทั้งหมดของระบบ เพิ่มผู้ใช้งาน และกำหนดสิทธิ์

### 2. ส่วนของนักศึกษา (Student Portal)
* **แจ้งเรื่องร้องเรียน**: สามารถสร้างฟอร์มแจ้งปัญหา เลือกประเภทเรื่องที่ร้องเรียน (เช่น การเรียน, หอพัก, ทุนการศึกษา) และอธิบายรายละเอียด
* **ติดตามสถานะ (Tracking)**: ตรวจสอบความคืบหน้าของเรื่องที่แจ้งไปได้แบบ Real-time (รอดำเนินการ, กำลังดำเนินการ, เสร็จสิ้น)
* **ประวัติการร้องเรียน**: ดูประวัติเรื่องร้องเรียนย้อนหลังทั้งหมดของตนเองได้

### 3. ส่วนของเจ้าหน้าที่และผู้ดูแลระบบ (Admin/Staff Portal)
* **แดชบอร์ด (Dashboard)**: แสดงสถิติและภาพรวมของเรื่องร้องเรียนทั้งหมดเพื่อการวิเคราะห์
* **ระบบจัดการเคส (Case Management)**:
  * ดูรายละเอียดและอัปเดตระดับความเร่งด่วน/สถานะการทำงาน
  * มอบหมายงานให้เจ้าหน้าที่ที่เกี่ยวข้อง (Assign to Staff)
  * บันทึกผลการดำเนินงาน (Progress Reports / Comments)
* **พิมพ์รายงานสรุป (Document Generation)**: สร้างและพิมพ์รายงานสรุปเคสในรูปแบบเอกสาร A4 ที่เป็นทางการ พร้อมฟอร์แมตที่อ่านง่าย
* **จัดการผู้ใช้งาน (User Management)**:
  * **ระบบจัดการนักศึกษา**: เพิ่ม/แก้ไข ข้อมูลนักศึกษา และรหัสผ่าน
  * **ระบบจัดการเจ้าหน้าที่**: เพิ่มเจ้าหน้าที่, กำหนดสิทธิ์, และกำหนดสังกัด (คณะ/สาขาวิชา)

### 4. การออกแบบ UI/UX ระดับพรีเมียม
* **Modern Interface**: ดีไซน์ที่สะอาดตา ทันสมัย เน้นการอ่านง่าย
* **Responsive Design**: รองรับการใช้งานทุกหน้าจอ (PC, Tablet, Mobile)
* **Twin-Layer Modal Architecture**: ระบบ Pop-up/Modal ที่ออกแบบมาเป็นพิเศษเพื่อล็อกหน้าจอพื้นหลังขณะใช้งานฟอร์ม ช่วยให้มีสมาธิกับการกรอกข้อมูล และลดปัญหาการแสดงผล

## 🛠️ โครงสร้างข้อมูลเบื้องต้น (Core Schema)
* `User`: ข้อมูลผู้ใช้งานทั้งนักศึกษาและเจ้าหน้าที่ (ผูกกับคณะและสาขา)
* `Complaint`: ข้อมูลเรื่องร้องเรียนแต่ละเคส
* `Progress`: การบันทึกความคืบหน้าของการแก้ปัญหา
* `Faculty / Major`: โครงสร้างคณะและสาขาวิชาอ้างอิง
